## JAM

<div style={{ margin: "0", display: "flex" }}>
  <div style={{ width: "30%", margin: "0 5%" }}>
    <p>JavaScript</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6" }}>
      Dynamic functionalities are handled by JavaScript.
      <br />
      There is no restriction on which framework or library you must use.
    </p>
  </div>
  <div style={{ width: "30%", margin: "0 5%" }}>
    <p>Markup</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6" }}>
      Websites are served as static HTML files.
      <br />
      These can be generated from source files, <br />
      such as Markdown, using a Static Site Generator.
    </p>
  </div>
  <div style={{ width: "30%", margin: "0 5%" }}>
    <p>APIs</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6" }}>
      Server side operations are abstracted into reusable APIs and <br />
      accessed over HTTPS with JavaScript.
      <br />
      These can be third party services or your custom function.
    </p>
  </div>
</div>

## BENEFITS

<div style={{ margin: "0 5%", display: "flex" }}>
  <div style={{ width: "40%", marginRight: "20%" }}>
    <p style={{ marginBottom: "10px" }}>Faster performance</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Serve pre-built markup and assets over a CDN
    </p>
    <p style={{ marginBottom: "10px" }}>More secure</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      No need to worry about server or database vulnerabilities
    </p>
    <p style={{ marginBottom: "10px" }}>Less expensive</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Hosting of static files are cheap or even free
    </p>
  </div>
  <div style={{ width: "40%" }}>
    <p style={{ marginBottom: "10px" }}>Better developer experience</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Front end developers can focus on the front end, without being tied to a
      monolithic architecture.
      <br />
      This usually means quicker and more focused development
    </p>
    <p style={{ marginBottom: "10px" }}>Scalability</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      If your product suddenly goes viral and has many active users, <br />
      the CDN seamlessly compensates
    </p>
  </div>
</div>

## BEST PRACTICES

<div style={{ margin: "0 5%", display: "flex" }}>
  <div style={{ width: "40%", marginRight: "20%" }}>
    <p style={{ marginBottom: "10px" }}>Content delivery network</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Since all the markup and assets are pre-built, they can be served via CDN.
      <br />
      This provides better performance and easier scalability.
    </p>
    <p style={{ marginBottom: "10px" }}>Atomic deploys</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Each deploy is a full snapshot of the site. This helps guarantee a
      consistent version of the site globally.
    </p>
    <p style={{ marginBottom: "10px" }}>Cache invalidation</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Once your build is uploaded, the CDN invalidates its cache. This means
      that your new build is live in an instant.
    </p>
  </div>
  <div style={{ width: "40%" }}>
    <p style={{ marginBottom: "10px" }}>Everything in version control</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Your codebase lives in Version Control System, such as Git. The main
      benefits are: change history of every file, collaborators and
      traceability.
      <br />
      This usually means quicker and more focused development
    </p>
    <p style={{ marginBottom: "10px" }}>Automated builds</p>
    <p style={{ fontSize: "20px", color: "#B6B6B6", marginTop: "5px" }}>
      Your server is notified when a new build is required, typically via
      webhooks. Server builds the project, updates the CDNs and the site is
      live.
    </p>
  </div>
</div>

      The modern web revolution starts prioritising the importance of
      performance, scalability and developer experience. The term JAMstack
      starts to be adopted by a wider group of developers and the first
      enterprise JAMstack projects are announced.
    </p>
